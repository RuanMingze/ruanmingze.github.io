**1\. 访问与加速：解决国内访问难题**

由于网络原因，直接访问 GitHub 经常会出现加载缓慢、图片无法显示甚至无法打开的情况。这里为你提供几种从简单到复杂的加速方案，你可以根据自己的需求选择。

**1.1 最简单的方案：在线加速转换**

如果你只是偶尔需要下载某个文件或者访问某个页面，不想修改系统设置，这是最适合你的方法。

**使用步骤：**

访问在线加速工具，例如 [GitWarp](https://www.gitwarp.com)

将你原本的 GitHub 链接（例如 https://github.com/username/repo）复制到输入框

点击转换按钮，工具会生成一个加速后的链接

使用新的链接访问，即可享受高速网络

这种方法无需安装任何软件，即开即用，非常适合临时使用。

**1.2 一劳永逸：修改 Hosts 文件**

如果你想长期稳定地加速 GitHub 访问，修改系统 Hosts 文件是最经典有效的方案。它的原理是直接指定 GitHub 相关域名的最优 IP 地址，绕过 DNS 解析的污染。

**操作步骤：**

获取最新的 Hosts 列表

访问开源项目 [GitHub520](https://github.com/521xueweihan/GitHub520)，这里会有人定期维护最新的 IP 地址。

复制页面中提供的最新 Hosts 内容。

修改系统 Hosts 文件

**Windows 系统**：

打开路径 C:\\Windows\\System32\\drivers\\etc\\

找到 hosts 文件，用记事本（或 VS Code）以管理员身份打开

将复制的内容粘贴到文件的最底部，保存。

**Mac/Linux 系统**：

打开终端，输入 sudo vim /etc/hosts

输入密码后，将内容粘贴进去，保存退出。

刷新 DNS 缓存

Windows：打开 CMD，输入 ipconfig /flushdns

Mac：输入 sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder

完成后，重新打开浏览器，你会发现 GitHub 的访问速度有了质的飞跃。

**1.3 小白友好：使用自动化工具**

如果你觉得手动修改 Hosts 太麻烦，而且担心 IP 过期，那么可以使用自动化工具。

**推荐工具：SwitchHosts**  
这是一款跨平台的免费工具，可以帮你一键管理 Hosts 配置，并且支持自动远程更新。

下载并安装 [SwitchHosts](https://oldj.github.io/SwitchHosts/)

添加一个新的配置，选择「远程」类型

URL 填入 https://raw.githubusercontent.com/521xueweihan/GitHub520/master/hosts

开启自动更新，工具会定期帮你拉取最新的 IP，全程无需手动操作。

**其他推荐工具：**

**FastGithub**：专门为 GitHub 加速的本地代理工具，开箱即用。

**DevSidecar**：开发者边车，不仅加速 GitHub，还能加速很多其他的开发源。

**2\. 账号注册与初始化**

解决了访问问题后，我们来注册一个属于自己的 GitHub 账号。

**2.1 注册账号**

打开浏览器，访问 [GitHub 官网](https://github.com)

点击页面右上角的 **Sign up**（注册）按钮。

填写注册信息：

**Username（用户名）**：这是你在 GitHub 的唯一标识，别人通过 github.com/你的用户名 就能访问你的主页。建议取一个简洁、专业的名字。

**Email（邮箱）**：填写你常用的邮箱，用于接收验证邮件和通知。

**Password（密码）**：设置一个强密码。

完成人机验证后，点击创建账户。

去你的邮箱里查收 GitHub 的验证邮件，点击验证链接，激活账号。

**2.2 完善个人资料**

注册完成后，建议先完善你的个人资料，这会让你的主页看起来更专业。

点击右上角的头像，选择 **Settings**（设置）

在 **Public profile**（公开资料）中，你可以：

设置你的昵称（Display name）

填写个人简介（Bio）：简单介绍一下你自己

填写所在地、公司、个人网站等信息

上传一个头像，让你的账号更有辨识度。

**3\. 核心功能入门：仓库、Star 与 Fork**

熟悉了账号之后，我们来了解 GitHub 最核心的几个概念。

**3.1 什么是仓库（Repository）**

仓库（Repository，简称 Repo）就是 GitHub 用来存放项目的地方。你可以把它理解为一个文件夹，里面存放着你项目的所有代码、文档、图片等文件。每个项目都对应一个独立的仓库。

**3.2 创建你的第一个仓库**

在网页端创建仓库非常简单：

登录后，点击右上角的 **+** 号，选择 **New repository**。

填写仓库信息：

**Repository name**：仓库的名字，比如 my-first-project

**Description**：可选，简单描述一下这个项目是做什么的。

**Visibility**：可见性。

**Public**：公开的，所有人都能看到。

**Private**：私有的，只有你和你授权的人能看到。

勾选 **Add a README file**，这样仓库创建好后会自动生成一个说明文件。

点击 **Create repository**，完成！

现在，你就拥有了自己的第一个 GitHub 仓库了。

**3.3 Star：收藏你喜欢的项目**

当你在浏览别人的项目时，如果遇到你喜欢或者觉得有用的项目，你可以点击项目右上角的 **Star** 按钮。

Star 就像是微信的 "点赞" 或者浏览器的 "收藏夹"。

对于项目作者来说，Star 数是衡量项目受欢迎程度的重要指标。

对于你来说，点击 Star 后，这个项目就会被加入到你的收藏列表里，以后你随时可以在自己的主页里找到它，方便回顾。

**3.4 Fork：复刻项目，自由修改**

如果你想参与一个开源项目，或者想基于别人的项目做一些自己的修改，你就需要用到 **Fork** 功能。

Fork 的意思是 "复刻"。当你点击别人仓库右上角的 **Fork** 按钮时，GitHub 会在**你自己的账号下**，自动复制一份这个仓库的完整副本。

**为什么要这么做？**  
因为你没有权限直接修改别人的仓库。但是 Fork 过来之后，这个副本就是你的了！你可以在这个副本里随意修改、添加文件，完全不会影响原来的项目。

**4\. 高效协作：Issue 与 Pull Request**

GitHub 不仅仅是一个存文件的地方，它更是一个强大的协作平台。Issue 和 Pull Request 就是协作的核心。

**4.1 Issue：反馈问题与讨论**

当你在使用别人的项目时，发现了一个 Bug，或者有一个新功能的建议，你就可以通过 **Issue** 来告诉作者。

Issue 可以理解为项目的 "留言板" 或者 "工单系统"。

**如何创建一个 Issue：**

进入目标项目的仓库页面。

点击上方的 **Issues** 标签页。

点击绿色的 **New issue** 按钮。

填写标题：简洁地描述你的问题，比如 "登录按钮在手机上无法点击"。

填写描述：详细说明你遇到的情况，最好附上截图或者错误信息，方便作者定位问题。

点击 **Submit new issue** 提交。

提交之后，项目的维护者就会收到通知，他们会看到你的反馈，并和你在这个 Issue 下面进行讨论。

**4.2 Pull Request：贡献代码**

如果你不仅想反馈问题，还想自己动手修复 Bug，或者添加新功能，然后把你的修改分享给原项目，这时候就需要用 **Pull Request**（简称 PR，拉取请求）。

这是参与开源贡献的标准流程，全程都可以在网页上完成：

**Fork 仓库**：首先，按照上一节的方法，把原项目 Fork 到你自己的账号下。

**修改代码**：在你自己 Fork 过来的仓库里，修改文件。你可以直接在网页上点击编辑按钮修改，改完后提交。

**发起 Pull Request**：

修改完之后，回到你自己的这个仓库页面，你会看到一个绿色的 **Compare & pull request** 按钮，点击它。

或者，你也可以回到原项目的页面，点击 **Pull requests** -> **New pull request**。

确认你要把自己的哪个分支的修改，合并到原项目的哪个分支。

填写 PR 的标题和描述，告诉作者你改了什么，为什么这么改。

点击 **Create pull request**。

这样，你就把你的修改请求发送给了原作者。作者会收到通知，他可以查看你具体改了哪些代码，如果他觉得你的修改很好，就会点击 **Merge** 按钮，把你的修改合并到原项目中。这样，你就成功为开源项目贡献了代码！

**5\. 个性化进阶：打造你的专属主页**

当你熟悉了基础操作后，你可能会想让自己的 GitHub 主页变得更酷、更个性化，成为你的线上简历。

**5.1 打造炫酷的个人主页 README**

GitHub 有一个非常酷的功能：如果你创建一个**和你的用户名完全同名**的仓库，那么这个仓库里的 README.md 文件，就会自动显示在你的个人主页上！

**操作步骤：**

点击右上角 **+** -> **New repository**。

**仓库名必须和你的 GitHub 用户名一模一样**。比如你的用户名是 zhangsan，那仓库名就必须是 zhangsan。

如果你填对了，GitHub 会自动提示你："You found a secret!"，说明你找对了。

勾选 **Add a README file**。

点击创建仓库。

创建好之后，你就可以编辑这个 README.md 文件了。通过 Markdown 语法，你可以在里面写自我介绍、展示你的技能、放上你的项目链接等等。

**5.2 让 README 动起来**

你还可以添加各种动态的小部件，让你的主页更炫酷：

**GitHub Stats 卡片**：自动统计你的 GitHub 数据，比如提交数、Stars 数、常用语言。

只需要在 README 里添加一行代码：\\\\

**技能徽章**：展示你会的编程语言、工具、框架，比如 \\\\

**动态访客计数器**：记录有多少人访问过你的主页。

这些小部件能让你的主页瞬间从普通的列表，变成一个非常专业的个人作品集，很多面试官在看简历之前，都会先逛逛你的 GitHub 主页哦。

**5.3 写好项目的 README**

对于你自己的每个项目仓库，一个好的 README.md 也至关重要。它是别人了解你项目的第一扇门。一个标准的 README 应该包含：

**项目标题与简介**：一句话说清楚这个项目是干嘛的。

**功能介绍**：列出项目的核心功能。

**安装 / 使用教程**：告诉别人怎么用你的项目。

**截图 / 演示**：放上项目的运行截图，直观展示效果。

**许可证**：说明这个项目的开源协议。
